# 内容创作模板

## 概览

本文件提供公众号内容创作中常用的模板，包括选题讨论、协作文档、长文转X等场景的标准化格式。

## 选题讨论模板

### 标准格式

当需要讨论选题时，提供3-4个选题方向，每个包含以下要素：

```plaintext
## 选题方案 [序号]

**标题**：<吸引人的标题>

**核心角度**：<一句话说明这个选题的核心观点/切入点>

**工作量评估**：⭐/⭐⭐/⭐⭐⭐（1-3星，3星最复杂）

**大纲**：
1. <一级标题>
2. <一级标题>
3. <一级标题>
...
（3-7个大标题）

**优劣势分析**：
- 优势：<为什么这个选题好>
- 劣势：<可能的挑战或不足>
```

### 给品牌方的大纲要求

- **字数要求**：<300字
- **风格**：口语化
- **结构**：只有一级标题，不要子标题

### 示例

```plaintext
## 选题方案1

**标题**：我花了一周测试了5款AI写作工具，发现了一个有趣的现象

**核心角度**：通过对比测试5款主流AI写作工具，发现它们在不同场景下的优劣，并给出选择建议

**工作量评估**：⭐⭐⭐

**大纲**：
1. 为什么要做这个测试？
2. 我测试了哪些工具？
3. 测试过程和方法
4. 每个工具的表现（重点）
5. 我的最终选择和理由

**优劣势分析**：
- 优势：实用性强，对比全面，有数据支撑
- 劣势：测试工作量较大，需要真实使用体验
```

## 协作文档模板

### 标准格式

当需要协调测试、配图等任务时，创建协作文档：

```plaintext
# [项目名] 协作清单

## 测试任务清单

- [ ] <任务1>
- [ ] <任务2>
- [ ] <任务3>

## 配图需求清单

| 序号 | 位置 | 内容 | 要求 | 状态 |
|------|------|------|------|------|
| 1 | 开头 | 产品logo | 高清 | 待制作 |
| 2 | 第3段 | 产品界面截图 | 标注关键功能 | 待制作 |
| 3 | 第5段 | 数据图表 | 数据准确 | 待制作 |

## 时间成本预估

| 任务 | 预计时间 | 备注 |
|------|----------|------|
| 信息搜索 | 2小时 | 包含竞品对比 |
| 测试体验 | 4小时 | 需要真实使用 |
| 初稿撰写 | 3小时 | 约3000字 |
| 审校优化 | 2小时 | 三遍审校 |
| **总计** | **11小时** | |
```

### 示例

```plaintext
# Notion AI 深度体验 协作清单

## 测试任务清单

- [ ] 注册并体验Notion AI
- [ ] 测试写作辅助功能（续写、扩写、改写）
- [ ] 测试翻译功能
- [ ] 测试总结功能
- [ ] 对比Copy.ai和Jasper

## 配图需求清单

| 序号 | 位置 | 内容 | 要求 | 状态 |
|------|------|------|------|------|
| 1 | 开头 | Notion AI logo | 高清，官方提供 | 已获取 |
| 2 | 第2段 | 界面截图 | 标注AI入口 | 待制作 |
| 3 | 第4段 | AI写作演示 | 展示续写效果 | 待录制 |
| 4 | 第5段 | 对比表格 | 3个工具对比 | 待制作 |
| 5 | 第7段 | 使用场景 | 实际工作场景 | 待拍摄 |

## 时间成本预估

| 任务 | 预计时间 | 备注 |
|------|----------|------|
| 信息搜索 | 2小时 | 查看官方文档和用户评价 |
| 测试体验 | 4小时 | 深度使用3天 |
| 初稿撰写 | 3小时 | 约3500字 |
| 审校优化 | 2小时 | 重点检查流畅度 |
| 配图制作 | 2小时 | 5张图 |
| **总计** | **13小时** | |
```

## 长文转X模板

### 标准结构（200-500字）

```plaintext
<开头30-50字：金句/数据/价值主张>

<展开100-300字：具体细节、个人经历>

<结尾20-30字：互动引导>
```

### 示例

```plaintext
用了100+个AI工具后，我发现效率工具的终极形态其实很简单。

最近一周，我把常用的Notion、Obsidian、Roam Research都卸载了，只保留了一个工具。为什么？因为我发现工具越少，效率越高。就像上次我和朋友聊的，"工具应该为你服务，而不是你为工具服务"。

你正在用几个效率工具？
```

## 爆款结构模板

### 问题-原因-方案-升华（教程类）

```plaintext
<开头：提出问题/痛点>
"你是不是也遇到过...？"

<原因分析>
为什么会出现这个问题？<具体原因分析>

<解决方案>
我是这样解决的：<具体步骤和方法>

<升华>
这背后其实是...<更深的洞察或原则>
```

### 故事-冲突-转折-洞察（经验类）

```plaintext
<故事开头>
说起来，<具体时间、地点、人物>...

<冲突出现>
但问题是...

<转折点>
然后幸运地发现...

<洞察收获>
这让我意识到...
```

### 颠覆认知-新视角-重建（观点类）

```plaintext
<颠覆认知>
大家都认为...但我的看法不一样。

<新视角>
为什么？因为...

<重建认知>
所以真正重要的是...
```

### 对比-反差-原因-启示（对比类）

```plaintext
<对比引入>
我对比了A、B、C几个...

<反差发现>
A很强，但B反而...

<原因分析>
为什么？因为...

<启示>
这给我们一个重要启示...
```

### 数据冲击-深挖-案例-升华（分析类）

```plaintext
<数据冲击>
这个数据还挺猛的...<具体数据>

<深度挖掘>
为什么这么高/低？<分析原因>

<案例支撑>
比如...<具体案例>

<升华>
这说明...<更深的洞察>
```
